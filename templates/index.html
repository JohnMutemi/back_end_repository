Relationships:

  • Users: The Users table contains information about all the users in the system, including their roles (doctor, patient, etc.).
  • Doctors: This table stores specific details about doctors, which are a subset of users.
  • Patients: This table contains details about patients, who are also a subset of users.
  • Appointments: The Appointments table tracks all appointments made between doctors and patients.
